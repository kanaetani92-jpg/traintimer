# Stage 44：スマホのタイマー画面を軽くする 検証メモ

## 実施日
2026-06-12

## 実施内容

iPhone・Androidスマホ縦画面で、タイマー画面が見やすくなるようにCSSを中心に調整した。

主な変更は以下。

- スマホ縦画面では、`body.page-timer-active` と `.main-content` の `overflow:hidden` による見切れを避け、縦スクロール可能な構成へ変更。
- `100dvh` と `env(safe-area-inset-bottom)` を使い、固定下部ナビと操作ボタンの重なりを避ける下余白を追加。
- `#pageTimer` をスマホ縦で1カラム、自然な順序に固定。
- 線路表示エリアをスマホ縦向けに大きめの `clamp()` 高さへ調整。
- 横線路・縦線路・円形線路ごとに、スマホ縦向けの高さを補正。
- 現在区間・次の駅カードを読みやすい高さと文字サイズに調整。
- 操作ボタンを線路の下のカード内に分離し、Grid/Flexで配置。
- 主操作ボタンは48px程度、359px以下では44px以上を確保。
- 「つづきから」と「はじめから」は横並びを維持。
- ＋1分・＋5分は主操作と分け、2列で表示。
- タイマー画面に運行中の電車リストは表示せず、小さな「運行中の電車：○本」表示のみ維持。

## 動作確認の観点

CSS上で以下の状態を確認した。

- iPhone/Androidスマホ相当の幅では、`#pageTimer` が1カラムで表示される。
- スマホ縦では、タイマー画面が以下の順序になる。
  1. 赤いヘッダー
  2. 線路切替
  3. 線路表示エリア
  4. 現在区間・次の駅
  5. はじめる／一時停止／つづきから／はじめから
  6. ＋1分・＋5分
  7. 下部ナビ
- 操作ボタンは `position:absolute` や `top:%` ではなく、Grid/Flexで配置される。
- 固定下部ナビ用に、スマホ縦の `.main-content` に十分な下部余白を設定している。
