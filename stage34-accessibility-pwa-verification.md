# Stage 34 操作性・アクセシビリティ・PWA確認メモ

## 実装・調整内容

- アプリバージョンを `1.17.3` に更新しました。
- Service Worker キャッシュ名を `train-timer-v1.17.3` に更新しました。
- キーボード操作の説明を実装に合わせ、`R` を「はじめから」として整理しました。
- `Space` は開始・一時停止、`R` ははじめから、`Esc` は開いているモーダルやパネルを閉じる操作として維持しました。
- ロック中に `Esc` を押した場合は、ロックを解除せず、長押し解除の進行をキャンセルしてロック中であることを通知するようにしました。
- タッチ端末では主要ボタン・ナビ・メニュー・保存系ボタンが 48px 以上になるように補強しました。
- マウス端末では hover 時の視覚フィードバックを補強しました。
- フォーカスリングを見やすくし、ボタン、入力欄、リンク、スイッチで見失いにくくしました。
- 文字サイズを特大にしたとき、カードや一覧が無理に横並びになりすぎないように補強しました。
- 高コントラスト時のフォーカスリングと境界線を補強しました。
- 動きを減らす設定時は、hover/active 由来の移動も抑えるようにしました。
- いたずら防止ロック中は、背面のタイマー表示を見せたまま、背面操作を受け付けないようにしました。
- 3秒長押し解除ボタンと進捗バーの視認性を維持しました。
- Wake Lock 判定を `canUseWakeLock()` に集約し、非対応環境でもエラーで止まらないようにしました。

## 確認した実装上の対応関係

| 項目 | 対応 |
|---|---|
| タッチ端末のボタン高 | `@media (pointer: coarse)` で 48px 以上に補強 |
| マウス端末の hover | `@media (hover: hover) and (pointer: fine)` で補強 |
| Space | 開始・一時停止 |
| R | はじめから |
| Esc | モーダル・作成画面・設定パネル・メニューを閉じる。ロック中は長押し解除をキャンセル |
| フォーカスリング | `:focus-visible` を強化 |
| 高コントラスト | 境界線・フォーカス色を補強 |
| 動きを減らす | transition / transform を抑制 |
| ロック中の誤操作防止 | 背面UIの pointer-events を止め、解除ボタンだけ操作可能 |
| Wake Lock非対応 | `canUseWakeLock()` で判定し、非対応文言を表示 |
| PWAキャッシュ | `train-timer-v1.17.3` に更新 |
