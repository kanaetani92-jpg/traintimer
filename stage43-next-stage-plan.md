# 第2段階以降の修正方針

## 第2段階：スマホのタイマー画面を軽くする

- `body.page-timer-active` の `overflow:hidden` と `height:100dvh` を再確認する。
- 線路エリア、現在区間、操作、下部ナビの合計高さを端末別に再配分する。
- タイマー画面は運行中リストを表示せず、小さな本数表示だけにする。

## 第3段階：線路表示エリアの端末別調整

- 横線路、縦線路、円形線路の必要高さを別々に管理する。
- `%` 指定は駅・電車・進行位置だけに限定する。
- `height:100% !important` と `overflow:hidden !important` の影響を弱める。

## 第4段階：操作ボタンと下部ナビの重なり対策

- 操作ボタンはGrid/Flexの専用コンテナにまとめる。
- 下部ナビがある端末では、ページ末尾に十分なsafe-area余白を入れる。
- 経路編集の保存ボタン、新しい電車追加パネルの保存ボタンも対象にする。

## 第5段階：新しい電車追加フォームの見やすさ改善

- 作り方、電車名と時間、駅数/1駅間、線路と音、プレビュー、保存のカード構成に分ける。
- スマホでは1カラム、PCでは最大幅を設定する。
- 保存ボタンを下部ナビやキーボードに隠れにくくする。

## 第6段階：管理画面の情報整理

- My設定は「すぐ使う → 運行中 → 新しい電車 → 登録した経路 → データ管理」の順にする。
- 削除・初期化は下部に控えめに置く。
- 経路編集と設定ページはカード単位で読みやすくする。

## 第7段階：PC・タブレット横の2カラム再調整

- 右操作パネルの最小幅を確保する。
- PCの全体最大幅を設定し、入力欄が伸びすぎないようにする。
- タブレット横はタッチ端末としてボタンサイズを維持する。

## 第8段階：端末別統合確認・ZIP納品

- 6画面幅で、タイマー・My設定・新規作成・経路編集・設定を確認する。
- 横線路・縦線路・円形線路を確認する。
- localStorage、PWA、ロック、複数電車を確認する。
