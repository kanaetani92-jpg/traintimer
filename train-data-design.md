# 第1段階：運行中の電車データ設計

## 目的

この段階では、既存のでんしゃタイマーの見た目と1本表示タイマーの動作を維持しながら、将来的に複数の電車を扱える保存データ形式を追加します。

## 用語

- 同時に走る電車を「運行中の電車」と呼びます。
- アクティブタイマー → いま見る電車
- タイマーリスト → 電車リスト
- 時間のまとまりは「区間」または「駅間」と呼びます。
- 保存済みの設定は「よく使う路線」または「登録した経路」と呼びます。

## 保存データ

localStorage の `trainTimerAppData` に、従来の現在設定に加えて以下を保存します。

```json
{
  "schemaVersion": 8,
  "activeTrainId": "train-...",
  "trains": [
    {
      "id": "train-...",
      "name": "いま見る電車",
      "status": "idle",
      "settings": { "unitMinutes": 5, "trackShape": "horizontal" },
      "stations": [],
      "segments": [],
      "totalMinutes": 30,
      "remainingMs": 1800000,
      "soundSettings": {},
      "createdAt": "ISO日時",
      "updatedAt": "ISO日時"
    }
  ]
}
```

## 旧データ対応

`trains` が存在しない旧データを読み込んだ場合は、現在の設定から `いま見る電車` を1本作成します。

## この段階で実装しないこと

- 複数電車の同時カウント
- 電車リストの大きなUI追加
- 新しい電車の追加画面
- 画面ロック
- Wake Lock


## 第2段階で追加した一覧UI

My設定ページに「運行中の電車」セクションを追加しました。

各カードに表示する内容は以下です。

- 電車名
- 残り時間
- 状態
- 線路形状
- 駅数
- 表示する / 表示中
- 削除

この段階では、複数の電車の同時カウントはまだ行いません。表示中ではない電車は、保存された残り時間と状態を一覧表示するだけです。

最後の1本を削除した場合は、初期設定の「いま見る電車」を自動作成し、アプリが空にならないようにします。


## 第3段階で追加した設計

- My設定ページの「運行中の電車」に「今の経路を電車として追加」ボタンを追加しました。
- ボタンを押すと名前入力ダイアログを表示します。
- 初期名は「朝の支度号」「勉強号」「休憩号」「お片付け号」「電車2」の順に候補を出します。
- 追加される電車は、現在表示中の経路設定をコピーします。
- 追加した電車は開始前の状態にします。
- 追加しても `activeTrainId` は変更せず、タイマー画面の表示対象は切り替えません。
- 追加後はMy設定ページの「表示する」ボタンで表示対象を切り替えます。
- 指定された除外路線名は、候補にも初期データにも含めていません。


## 第4段階追記

- My設定ページの各電車カードに、はじめる／一時停止／つづきから／はじめから／表示する／削除の操作を追加しました。
- 表示中ではない電車も、運転中であれば裏で残り時間を更新します。
- 到着した電車は「到着」と表示し、複数本が同時に到着した場合はまとめて通知する設計にしました。
- タイマー画面は、引き続き1本の電車を大きく表示する構成を維持しています。

## 第5段階追記

- My設定ページに「新しい電車を追加」ボタンを追加しました。
- 作成画面はボトムシートとして表示します。
- 入力項目は、電車名、全体時間、駅の数、1単位の分数、線路の形、音です。
- 駅名は、駅数に合わせて「スタート」「駅1」「駅2」「ゴール」の形式で自動生成します。
- 区間は駅数に合わせて自動生成し、全体時間に合うように区間時間を配分します。
- 入力例として、勉強25分号、休憩5分号、お片付け10分号、朝の支度15分号、読書15分号を用意しました。
- 作成後にタイマー画面へ切り替えるかどうかを、作成画面内のチェックで選べます。
- 初期状態では自動切替しません。


## 第6段階：すぐ使う路線

My設定ページに「すぐ使う」セクションを追加しました。ここでは日常場面や支援場面で使いやすい路線を、ボタン1つで「運行中の電車」へ追加します。追加された電車は開始前の状態で保存され、自動出発はしません。通常は表示中の電車も切り替えません。

### 追加される路線

- 休憩5分号：スタート → 休憩駅 → ゴール
- お片付け10分号：スタート → 前半 → 後半 → ゴール
- 朝の支度15分号：スタート → 準備 → 確認 → ゴール
- 読書15分号：スタート → 前半 → 後半 → ゴール
- 勉強25分号：スタート → 集中1 → 集中2 → まとめ → ゴール

指定の除外路線は含めていません。

## 第7段階で追加した設定

`preferences` に以下を追加しました。

- `childLockEnabled`: いたずら防止ロックのオン・オフ。
- `keepAwakeEnabled`: 運転中に画面をつけたままにする設定。

旧データにこれらの項目がない場合は、どちらも `false` として扱います。
